1. 전역 컴포넌트, 지역 컴포넌트
=> app.component를 사용해서 컴포넌트를 등록하면 전역 컴포넌트가 됩니다.
=> 어디에서나 쓸 수 있다는 장점이 있지만 app 루트 설정 코드가 장황해집니다.
=> 또한 앱의 진입점에서 모든 컴포넌트를 읽어서 등록해야 하기에 성능에 악영향을 미칩니다.

=> 컴포넌트 script 설정 객체에서 components를 사용해서 지역 컴포넌트를 설정할 수 있습니다.
=> 지역 컴포넌트란 해당 컴포넌트에서만 사용 가능한 컴포넌트를 의미합니다.
=> components : {
    key : value
}
=> key를 카멜 케이스로 사용하면 template에 적용할 떄
 카멜 케이스로 사용해도 됩니다. (이 떄 셀프 클로징 태그 가능), 또한 자동으로 케밥 케이스를 사용해도 됩니다.
=> key가 케밥 케이스라면 templage에 적용할 떄 케밥 케이스만 사용해야 합니다.


2. Scoped Style
=> style에서 스타일을 정의하면 전역으로 설정되어 원치 않는 효과를 낳을 수 있습니다.
=> <style scoped>를 통해 해당 vue의 template에 스타일로 제한할 수 있습니다.
