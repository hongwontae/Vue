1. Vuex
=> 전역 상태를 관리하기 위한 라이브러리입니다. (상태 => 반응형 데이터)

=> 지역상태 : 하나의 컴포넌트에서만 사용하는 상태입니다.
=> 전역상태 : 여러 컴포넌트에 걸쳐 사용되는 상태입니다.


2. Vuex를 안쓰고 전역 상태를 사용할 떄 문제점
2-1. 하나의 컴포넌트에서 모든 데이터와 데이터 변경 메서드를 넣어야 합니다.
2-2. Provide, Inject는 반응성 관련된 복잡한 문제를 일으킬 수 있습니다.


3. Vuex의 장점
3-1. 컴포넌트에 상태 로직을 줄일 수 있습니다.
3-2. 상태에 대해 예상이 가능합니다.
3-3. 에러 가능성이 낮아집니다.
=> Vuex가 존재하면 상태 관리를 맡기 떄문에 컴포넌트 크기가 줄고, 예측 가능하고 이로 인해 오류도 적어집니다.
