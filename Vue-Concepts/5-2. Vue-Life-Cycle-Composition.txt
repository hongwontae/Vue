1. setup() (beforeCreate, created가 사라지고 setup이 이를 대신합니다.)
=> Vue3의 Composition API에서 가장 먼저 실행됩니다.
=> data(), methods(), computed()등이 초기화되지 않은 상태입니다.
=> this를 사용할 수 없고, 주로 반응형 데이터를 설정하는 곳입니다.


2. onBeforeMount()
=> DOM이 생성되기 직전에 실행됩니다.
=> 아직 template이 렌더링되지 않았습니다.


3. onMounted()
=> DOM이 생성된 후 실행됩니다.
=> 이 시점에서는 템플릿이 렌더링 완료되었습니다.
=> DOM을 직접 조작해야 한다면 이 훅에서 실행하는 것이 좋습니다.


4. onBeforeUpdate()
=> 반응형 데이터가 변경되었지만, DOM 업데이트가 되기 전 실행됩니다.
=> 상태는 바뀌었지만 화면에는 아직 반영되지 않은 상태입니다.


5. onUpdated()
=> 반응형 데이터가 변경되고, DOM이 업데이트된 후 실행됩니다.
=> 데이터 변경으로 인해 템플릿이 다시 렌더링된 후 실행됩니다.


6. onBeforeUnmount()
=> 컴포넌트가 파괴되기 직전에 실행됩니다.
=> 타이머 제거, 웹소켓 종료 등 정리(clean-up) 작업을 수행할 때 유용한 라이프 사이클 훅입니다.


7. onUnmounted()
=> 컴포넌트가 제거된 후 실행됩니다.
=> DOM에서 완전히 삭제된 이후 실행됩니다.
=> 이벤트 리스너 제거, 타이머 제거 등 정리 작업을 수행합니다.
