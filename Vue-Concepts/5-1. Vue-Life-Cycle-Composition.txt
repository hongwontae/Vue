1. beforeCreate()
=> 컴포넌트가 생성될 때 가장 먼저 실행되는 훅입니다.
=> 아직 data(), computed, methods가 초기화되지 않았습니다.
=> this를 사용해도 data()의 값에 접근할 수 없습니다.


2. created()
=> data(), computed, methods() 등이 초기화된 상태에서 실행됩니다.
=> 아직 template이 렌더링되지 않았으므로, DOM에 직접 접근할 수 없습니다.

** 1,2를 생성단계라고 부릅니다.


3. beforeMount()
=> 가상 DOM이 생성되었지만, 아직 실제 DOM에 적용되지 않았습니다.


4. mounted()
=> 실제 DOM이 생성되고, 가상 DOM이 실제 DOM에 적용되었습니다.

** 3,4를 생성 마운트 단계라고 부릅니다.


5. beforeUpdate()
=> 상태가 변경되었지만, 아직 DOM이 업데이트되지 않은 상태입니다.


6. updated()
=> 상태가 변경되어 DOM이 업데이트된 후 실행됩니다.

** 5,6을 업데이트 단계라고 부릅니다.


7. beforeDestroy()
=> 컴포넌트가 파괴되기 전에 실행됩니다.
=> 이벤트 리스너 정리 등을 수행하기 좋습니다.	


8. destroyed()	
=> 컴포넌트가 완전히 제거된 후 실행됩니다.

** 7,8을 제거 단계라고 부릅니다.
